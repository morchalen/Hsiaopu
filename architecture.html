🔌DAO 数据访问对象

Room 框架自动实现的数据库访问接口，无需手写 SQL（@Query 除外）

ConversationDao 接口
Daos.kt 📋 复制
@Dao interface ConversationDao {

// 获取所有对话，按更新时间倒序
@Query("SELECT * FROM conversations ORDER BY updatedAt DESC")
fun getAllConversations(): Flow<List<ConversationEntity>>

// 根据 ID 查询对话
@Query("SELECT * FROM conversations WHERE id = :id")
suspend fun getConversationById(id: Long): ConversationEntity?

// 插入对话（Room 自动生成 SQL）
@Insert
suspend fun insertConversation(conversation: ConversationEntity): Long

// 更新对话（按主键更新）
@Update
suspend fun updateConversation(conversation: ConversationEntity)

// 删除对话（按主键删除）
@Delete
suspend fun deleteConversation(conversation: ConversationEntity)
}
注解使用规则
注解 是否需要手写 SQL 说明
@Insert ❌ 插入数据的 SQL 是固定的，Room 自动生成
@Delete ❌ 按主键删除，也是固定的
@Update ❌ 按主键更新，也是固定的
@Query ✅ 查询千变万化（条件、排序、联表），Room 猜不到
💡 Flow 的作用：

当数据库数据发生变化时，Flow 会自动发出新的数据，UI 可以实时响应更新，实现"数据库变化 → UI 自动刷新"的效果。

🔑 StateFlow 与 MutableStateFlow：
  • MutableStateFlow：私有可变流，ViewModel 内部使用，用于发送状态更新
  • StateFlow：公共只读流，暴露给 UI 使用，通过 asStateFlow() 转换
  • collectAsState()：Compose 组件中订阅 StateFlow，数据变化时自动触发重组

DatabaseModule 代码详解
DatabaseModule.kt 📋 复制
@Module @InstallIn(SingletonComponent::class) object DatabaseModule {

// 提供 AppDatabase 单例实例
@Provides
@Singleton
fun provideAppDatabase(
  @ApplicationContext context: Context
): AppDatabase {
  return Room.databaseBuilder(
    context,
    AppDatabase::class.java,
    "hsiaopu_db"
  ).build()
  }

// 提供 ConversationDao
@Provides
fun provideConversationDao(db: AppDatabase): ConversationDao = db.conversationDao()

// 提供 MessageDao
@Provides
fun provideMessageDao(db: AppDatabase): MessageDao = db.messageDao()

// 提供 ShellHistoryDao
@Provides
fun provideShellHistoryDao(db: AppDatabase): ShellHistoryDao = db.shellHistoryDao()
}
Hilt 注解说明
注解 作用
@Module 标记这是一个依赖注入模块
@InstallIn 指定模块安装到哪个组件（SingletonComponent 表示全局单例）
@Provides 标记提供依赖的方法，Hilt 会调用此方法获取实例
@Singleton 标记单例，整个应用生命周期内只创建一次
@Inject 标记需要注入的构造函数或字段
@HiltViewModel 标记 ViewModel，让 Hilt 自动注入依赖
@ApplicationContext 注入 Application 级别 Context，避免内存泄漏

doSendWithTools 核心代码流程
1
存储用户消息
将用户输入插入数据库，并更新 UI 状态显示用户消息
2
构建工具系统提示词
调用 buildToolSystemPrompt()，将工具定义（WiFi控制、蓝牙控制、系统查询等）注入到 system prompt 中
3
第一轮流式请求
调用 sendMessageStream()，AI 返回可能包含 [TOOL:action_name:param=value] 格式的工具指令
4
解析并执行工具指令
使用正则表达式 \[TOOL:([a-z_]+)(?::([^\]]*))?\] 匹配工具指令，通过 ShellCommandBus 发送给 ShellExecutor 执行
5
第二轮总结请求（可选）
如果有工具执行结果，将结果追加到消息列表，发起第二轮 AI 请求进行总结
6
持久化最终结果
将 AI 的最终回复（包含工具执行结果）插入数据库，并更新对话标题

ShellCommandBus 核心代码
ShellCommandBus.kt 📋 复制
@Singleton class ShellCommandBus @Inject constructor() {

// 命令发送通道（replay=0 避免重复发送）
private val _commands = MutableSharedFlow<String>(
  replay = 0,
  extraBufferCapacity = 20
)
val commands = _commands.asSharedFlow()

// 结果返回通道
private val _results = MutableSharedFlow<ShellResult>(
  replay = 0,
  extraBufferCapacity = 10
)
val results = _results.asSharedFlow()

suspend fun sendCommand(command: String) {
  _commands.emit(command)
}

suspend fun sendResult(result: ShellResult) {
  _results.emit(result)
}
}
