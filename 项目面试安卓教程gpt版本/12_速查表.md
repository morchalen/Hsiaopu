## 一句话解释

- Compose：用 Kotlin 函数描述 UI，状态变化后自动重组。
- ViewModel：保存 UI 业务状态，避免 Activity/Composable 过重。
- StateFlow：适合暴露当前 UI 状态给 Compose。
- Room：SQLite 的官方抽象，适合结构化本地数据。
- DataStore：现代键值存储，适合设置项。
- Hilt：帮你创建和管理对象依赖。
- Retrofit：用接口和注解描述 HTTP API。
- OkHttp：真正执行 HTTP 请求的客户端。
- Shizuku：用户授权后提供高权限系统能力。

## 常见追问短答

**为什么源码在 java 目录但文件是 kt？**

Android 默认源码目录名可以叫 `java`，并不限制只能放 Java。Gradle Kotlin 插件会编译 `.kt` 文件。

**为什么 Room 查询返回 Flow？**

数据库变化时 UI 可以自动收到新数据，比如新增消息后聊天页自动更新。

**为什么公开 `StateFlow` 而不是 `MutableStateFlow`？**

防止 UI 层直接修改状态，保证状态只由 ViewModel 控制。

**为什么要 Repository？**

隔离数据访问细节，让 ViewModel 不直接依赖 DAO。

**为什么工具调用有风险？**

因为 AI 输出会触发系统命令，必须校验参数、限制危险操作并要求用户确认。
