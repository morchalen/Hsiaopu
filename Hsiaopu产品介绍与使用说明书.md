# Hsiaopu — 极客专属 AI 工作台

## 产品介绍

### 一句话定位

**Hsiaopu** 是一款面向 Android 极客开发者的全功能 AI 工作台，集成多模型 AI 对话、代码审查、Shizuku Shell 终端、设备工具箱于一体，采用 Jetpack Compose + Hilt + Room 打造的现代化企业级架构。

---

### 产品愿景

在移动端 AI 助手泛滥的今天，大多数产品停留在"套壳聊天"层面。Hsiaopu 的使命是：**做一个真正属于开发者的生产力工具**——既能聊代码、审查架构，也能直接操控手机底层，让 Android 设备变成随身开发工作站。

---

### 核心功能矩阵

| 模块 | 功能 | 技术亮点 |
|------|------|----------|
| **AI 对话** | 多轮流式对话、Markdown 渲染、代码块高亮、多会话管理 | OkHttp SSE 流式解析、自研 Markdown 解析器、Room 持久化 |
| **代码审查** | 粘贴代码 → 6 维度 AI 分析 → 改进建议 | 独立 Prompt 工程、不污染对话历史 |
| **Shell 终端** | 自定义命令输入 + 20 个预定义系统命令、输出折叠/复制 | Shizuku 提权执行、实时 stdout/stderr 分离 |
| **设备工具箱** | 设备信息、网络状态、存储、电池、传感器 | Android 系统 API 直接读取 |
| **多 Provider** | DeepSeek / OpenAI 兼容 / Ollama 任意切换 | 策略模式 `AiProvider` 接口，插件化注册 |
| **对话导出** | Markdown / JSON 一键导出 | 格式化模板引擎 |
| **自适应布局** | 手机底部导航 / 平板侧栏双栏 | Material3 Adaptive + WindowSizeClass |
| **主题系统** | 明/暗/跟随系统 + 5 种强调色 | DataStore 持久化偏好 |

## 使用说明书

### 一、环境要求

| 项目 | 最低要求 |
|------|----------|
| Android 版本 | 7.0 (API 24) 及以上 |
| 推荐设备 | Android 12+ 设备 |
| Shizuku 功能 | 需安装 [Shizuku](https://shizuku.rikka.app/) 并激活 |
| 网络 | 需连接互联网（AI 对话功能） |

---

### 二、安装与初始化

1. 从 Release 页面下载 APK，安装到设备
2. 首次启动将看到 Splash 品牌页
3. 进入设置页，配置 AI 服务：

   **DeepSeek 用户**：
   - 选择 Provider 为 "DeepSeek"
   - 填入 API Key（从 [platform.deepseek.com](https://platform.deepseek.com) 获取）
   - 默认 Endpoint 无需修改

   **Ollama 用户**：
   - 选择 Provider 为 "OpenAI Compatible"
   - Endpoint 填入 `http://<你的IP>:11434/v1`
   - API Key 可填任意值（如 `ollama`）
   - Model 填入你的模型名（如 `qwen2.5:7b`）

4. 如需 Shell 功能，安装并激活 Shizuku，在 Shizuku App 中授权 Hsiaopu

---

### 三、对话功能

#### 3.1 创建新对话
- 点击侧栏底部 "+" 按钮
- 对话标题自动从首条消息截取

#### 3.2 发送消息
- 底部输入框输入内容，点击发送按钮（或键盘回车）
- 输入框上方有 6 个快速指令胶囊：`解释代码` `翻译` `总结` `建议` `写代码` `头脑风暴`
- 点击任一胶囊自动填入对应 Prompt 前缀

#### 3.3 消息操作
- **长按消息** → 自动复制内容到剪贴板
- **Markdown 渲染**：支持标题、粗体、代码块（带语法高亮）、行内代码、列表
- **流式输出**：AI 回复以打字机效果逐字显示，带有闪烁光标动画
- **时间戳**：每条消息右侧显示 `HH:mm` 发送时间
- **回到底部**：手动上翻历史消息时，右下角出现 FAB 按钮，点击回到最新消息

#### 3.4 会话管理
- 点击侧栏图标（左上角）打开对话列表
- **搜索**：顶部搜索框实时过滤对话
- **重命名**：点击对话旁的编辑图标，输入新名称
- **删除**：左滑对话 → 红色删除按钮；或点击垃圾桶图标 → 确认弹窗
- **切换对话**：点击任意对话即可切换

#### 3.5 导出对话
- 对话页面顶部菜单 → 选择导出格式：
  - **Markdown**：适用于文档归档
  - **JSON**：适用于数据处理

---

### 四、代码审查

1. 底部导航切换到"审查" Tab
2. 在文本框中粘贴需要审查的代码
3. 点击"开始审查"按钮
4. AI 将从以下 6 个维度分析：
   - 代码结构与可读性
   - 潜在 Bug 与边界情况
   - 性能优化建议
   - 安全性审查
   - 最佳实践对照
   - 改进建议与示例代码
5. 审查结果独立展示，**不会**混入正常对话历史

---

### 五、Shell 终端

#### 5.1 前提条件
- Shizuku 已安装并运行
- 已在 Shizuku 中授权 Hsiaopu
- 页面顶部状态指示器显示"Shizuku 已连接"（绿色）

#### 5.2 使用预定义命令
- 分类标签栏：`System` `Process` `Network` `Storage` `Package` `HW`
- 点击分类切换，下方显示该分类的命令胶囊
- 点击胶囊 → 命令自动填入输入框 → 点击执行

| 分类 | 预定义命令 |
|------|-----------|
| **System** | CPU Info, Memory, Uptime, Kernel, Prop |
| **Process** | Top 10, PS, OOM |
| **Network** | DNS, Routes, ARP, TCP |
| **Storage** | Mount, DF, Block |
| **Package** | Packages, Services, Battery |
| **HW** | Sensors, Input, Display, Audio |

#### 5.3 自定义命令
- 在 `$` 提示符后的输入框中输入任意 Shell 命令
- 点击绿色播放按钮执行
- 执行结果以卡片形式展示：
  - 绿色圆点 + 命令名 + 退出码 `[0]`
  - 可折叠/展开 stdout 输出
  - 红色错误信息单独显示
  - 点击复制图标将输出复制到剪贴板
- 历史记录保留在列表中，支持滚动查看

#### 5.4 状态说明
| 状态 | 含义 |
|------|------|
| 绿色 "Shizuku 已连接" | 可正常执行命令 |
| 红色 "无权限" | 需在 Shizuku App 中授权 |
| 红色 "Shizuku 未连接" | Shizuku 服务未启动 |

### 九、常见问题

**Q: 消息会丢失吗？**
A: 所有对话和消息通过 Room 数据库本地持久化存储，不会丢失。

**Q: API Key 安全吗？**
A: API Key 通过 Android DataStore 加密存储，仅本地使用，不上传任何第三方。

**Q: Shizuku 是什么？为什么需要它？**
A: Shizuku 是一个开源的 Android 权限管理框架，允许普通应用以 ADB/root 级别权限执行系统命令。Shell 终端功能依赖 Shizuku 提权。

**Q: 支持哪些 AI 模型？**
A: 理论上支持所有兼容 OpenAI Chat Completions API 的服务，包括但不限于：DeepSeek、Ollama、vLLM、OpenRouter、Groq、Together AI 等。

## 五大师点评（恳请指正）

我们真诚地恳请以下五位前辈对 Hsiaopu 提出点评指正——我们想要开发一款面向极客的 App，是否能够成功？

**Steve Jobs**（苹果联合创始人，已故）："极客工具不是功能越多越好，而是每一处都恰到好处。Shizuku Shell 终端的概念很大胆，但你们要记住——用户不会因为你能执行 20 个系统命令而爱上你，他们会因为那个命令执行动画的 0.3 秒恰到好处而爱上你。Design is not just what it looks like, it's how it works. 把体验打磨到极致，而不是堆砌功能。"

**Tim Cook**（苹果 CEO）："隐私和本地优先的策略值得肯定，DataStore 加密存储 API Key 是负责任的做法。但我建议你们加入更完善的无障碍支持——VoiceOver/TalkBack 适配、足够的色彩对比度、动态字体缩放。好的产品服务于所有人，而不仅仅是极客。另外，考虑一下 App Store 的审核标准——Shizuku 这类提权功能在 iOS 生态中是不可行的，但 Android 的开放性给了你们独特的差异化空间。"

**雷军**（小米集团创始人）："Are you OK？这个项目让我想起当年我们做 MIUI 的初心——为发烧友而生。Hsiaopu 的多 Provider 架构、自适应布局、Shizuku 集成，这些都很硬核。但我想问：你们有没有考虑过普通用户？一个极客工具，最终能不能让 100 万用户用起来？建议你们加一个「新手模式」，一键配置 DeepSeek，零门槛上手。把极客的深度藏起来，把简单留给用户——这才是做产品的智慧。"

**任正非**（华为创始人）："技术架构扎实，分层清晰，这是好的工程素养。但我要提醒你们：不要为了炫技而炫技。AI 对话、代码审查、Shell 终端、设备工具——四个模块之间的逻辑关系是什么？它们能否形成一个闭环的工作流？比如：用户用 Shell 抓取日志 → 用 AI 分析日志 → 用代码审查生成修复方案 → 再回到 Shell 执行。把孤立的点连成线，才是真正的生产力工具。另外，建议加入离线 AI 能力（如端侧模型），在无网络环境下也能工作，这是华为一直坚持的方向。"

**罗永浩**（锤子科技创始人）："坦白讲，这是我这几年见过的大学生项目中，比较有想法的一个。但你们得明白一个道理——做产品不是做作业，不是为了交差了事。Hsiaopu 的交互细节还有打磨空间：那个打字机动画的闪烁频率能不能再舒服一点？消息气泡的圆角半径是不是该区分用户和 AI？代码块的字体回退到系统默认时有没有考虑过等宽字体缺失的情况？这些细节才是决定一个产品能不能「用起来爽」的关键。不过话说回来，这帮年轻人比我们当年强多了——至少他们真的写出来了，而不是光在 PPT 上画饼。干得漂亮，但别骄傲，继续打磨。"
